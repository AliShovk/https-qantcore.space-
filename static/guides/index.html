Обычный RAG отвечает на один запрос. Agentic RAG планирует поиск, переформулирует запросы, проверяет результаты и итерирует. Полный туториал с LangGraph.

Продвинутый ⏱ 16 мин
📋 Показать код / инструкцию
Архитектура Agentic RAG: граф из 5 узлов: Plan (анализирует вопрос → составляет план поиска), Retrieve (ищет в векторной БД + web search), Grade (оценивает релевантность каждого документа), Generate (пишет ответ на основе отфильтрованных документов), Reflect (проверяет полноту → если недостаточно → возврат к Plan). Код LangGraph: graph = StateGraph(AgentState); graph.add_node('plan', plan_node); graph.add_node('retrieve', retrieve_node); graph.add_conditional_edges('grade', should_continue, {'yes': 'generate', 'no': 'plan'}).

Создаём свой MCP-сервер за 20 минут

Python MCP-сервер с инструментами для базы данных

Создайте свой MCP-сервер на Python с инструментами для БД. Работает с Claude Desktop, Cursor, Continue. Полный код и деплой.

Средний ⏱ 20 мин
📋 Показать код / инструкцию
Шаг 1 — установка: pip install mcp. Шаг 2 — сервер: from mcp.server import Server; server = Server('db-tools'); @server.list_tools() → [Tool('query', 'Run SQL query', {'sql': str})]; @server.call_tool() → execute_query(args['sql']). Шаг 3 — конфиг Claude Desktop: добавить в claude_desktop_config.json: {'mcpServers': {'db-tools': {'command': 'python', 'args': ['server.py']}}}. Результат: Claude может выполнять SQL-запросы к вашей БД прямо из чата.

Observability для AI-агентов: tracing и метрики

LangSmith, Arize Phoenix, OpenTelemetry для агентов

Агенты недетерминированы — нужно видеть что они делают. Трассировка каждого шага, метрики, алерты. Production-уровень observability.

Продвинутый ⏱ 14 мин
📋 Показать код / инструкцию
Стек observability: 1) LangSmith — tracing для LangChain/LangGraph. os.environ['LANGCHAIN_TRACING_V2']='true' и все вызовы автоматически трассируются. 2) Arize Phoenix — OpenTelemetry-native. from phoenix.otel import register; register(project_name='my-agents'). 3) Кастомные метрики: @traceable(name='agent_step') декоратор для каждого шага агента. Ключевые алерты: latency > 3× baseline, success rate < 80%, cost per task > бюджет. Дашборд: Grafana + Prometheus для реального времени.

Модели деплоя AI-агентов: сравнительный обзор

Lambda, EC2, K8s, Modal, Fly.io — что выбрать

Сравнение 5 моделей деплоя для AI-агентов: холодный старт, масштабирование, стоимость. Практические рекомендации для каждого сценария.

Средний ⏱ 13 мин
📋 Показать код / инструкцию
1) Serverless (Lambda/Cloud Run): холодный старт 500ms-3s, pay-per-use. Идеально: редкие задачи, event-driven. 2) EC2/VM: холодный старт 0 (always-on), от $20/мес. Идеально: постоянная нагрузка, GPU-модели. 3) Kubernetes: автоскейлинг, canary deploy, от $50/мес (минимальный кластер). Идеально: сложные multi-agent системы. 4) Modal: Python-native serverless с GPU. @app.function(gpu='A10G'). Идеально: ML-задачи. 5) Fly.io: edge deployment, автоскейлинг. Идеально: low-latency агенты. Правило: прототип → Modal/Fly. Продакшен → K8s. Бюджетно → EC2.
